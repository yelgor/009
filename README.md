# 009 Laboratory Room

Короткий звіт по проєкту: як запустити і що вже реалізовано.

## Як запустити

### 1. Встановити залежності
```bash
npm install
```

### 2. Запустити API (json-server)
```bash
npm run api
```

API працює на `http://localhost:3001`.

### 3. Запустити фронтенд
```bash
npm start
```

Фронтенд відкриється на `http://localhost:3000`.

Важливо: потрібно запускати і API, і фронтенд (у двох терміналах).

## Реалізований функціонал

### Головна сторінка
- Hero-блок з кнопкою Learn More.
- Плашка (модальне вікно) з коротким описом лабораторії.
- У модалці відображається фото `real.jpeg` та заголовок `Місце чудес ✨`.

### Обладнання
- Список доступного обладнання.
- Сторінка деталей обладнання.
- Додавання обладнання в кошик.

### Реєстрація та вхід
- Реєстрація тільки з поштою домену `@ucu.edu.ua`.
- Перевірка, що email не дублюється.
- Після реєстрації користувач входить автоматично.
- Після входу/реєстрації повернення на попередню сторінку.
- У навбарі є кнопка виходу з акаунта.

### Кошик
- Кошик зберігається у `localStorage` (не зникає після перезавантаження).
- Оформлення позики доступне тільки для авторизованого користувача.
- Email в оформленні береться з акаунта (вручну не вводиться).
- Дата повернення: можна вибрати тільки сьогодні або майбутню дату.

### Мої позики
- Показуються тільки позики поточного залогіненого користувача.
- Якщо користувач вийшов, позики не відображаються.
- Кожен елемент позики відображається окремим рядком.
- При натисканні на видалення є підтвердження: `Так / Ні`.

### Статистика позик
- Всього видів обладнання: рахується за унікальними категоріями.
- Всього одиниць обладнання: сума кількостей по елементах.
- Найближча дата повернення: мінімальна дата серед активних позик.

## Технології
- React (CRA, `react-scripts`)
- React Router
- CSS Modules
- json-server (локальне API)
